# roasting 임원용 가이드

> `/roasting`은 화이트칼라 산출물을 5명이 함께 다듬는 Claude Code 플러그인입니다.
> 이메일부터 이사회 보고서까지, 63개 케이스 중 알맞은 것을 자동으로 골라 합격선 9.5점에 닿을 때까지 다듬습니다.

---

## 1. 시작 (5분)

**설치 (두 줄):**

```bash
/plugin marketplace add airoasting/roasting
/plugin install roasting@airoasting
```

**첫 호출 예시:**

```
/roasting 거래처 부장한테 8월 12일까지 회신 달라는 메일
```

이 한 줄을 넣으면 `/roasting`이 알맞은 케이스를 찾고, 캐스팅된 5명이 곧바로 작업에 들어갑니다.
결과는 Claude Code 대화창에서 확인할 수 있습니다.

---

## 2. 이게 무엇인가

세 가지가 맞물려 돌아갑니다.

| 구성 요소 | 역할 |
|---|---|
| **5color 사이트** | 63개 케이스 정의 원본 (airoasting.github.io/5color) |
| **slide_library** | PPT 케이스용 슬라이드 템플릿 35종 |
| **/roasting** | 자동 분류 + 5인 팀 실행 엔진 |

자연어로 요청하면 `/roasting`이 63개 케이스 중 가장 가까운 것을 찾습니다. 그 케이스에 맞춰 5명의 캐스팅이 정해집니다. 한 명은 산출물을 짓고, 나머지 네 명은 각자 다른 관점에서 평가합니다.

## 4. 5-Color 방법론

5명이 산출물 하나를 함께 만들고 평가합니다.

| 역할 | 담당 관점 | 한 줄 |
|---|---|---|
| **BLACK** | 수행자 | 케이스에 맞는 전문가 페르소나로 산출물을 직접 짓습니다 |
| **RED** | 이성·논리 | 의도가 한 줄로 전달되는지, 논리 구조가 단단한지 봅니다 |
| **SILVER** | 분야 전문가 | 해당 문서 유형의 구조와 관행에 맞는지 짚습니다 |
| **BLUE** | 공감·톤 | 읽는 사람 입장에서 톤과 분량이 적절한지 봅니다 |
| **GOLD** | 독자 시나리오 | 실제 수신자가 이 문서를 받는 순간을 시뮬레이션합니다 |

**합격 기준:**
- RED·SILVER·BLUE 3인 평균 9.5 이상
- GOLD 체류도 9.5 이상
- 최대 4라운드 반복

4명의 점수가 0.5점 이상 벌어지면 그 자리에서 짧게 토론합니다. 새 점수를 매겨 다음 라운드로 넘어갑니다.

---

## 5. PPT 케이스: slide_library 결합

임원 PPT, 영업 덱, 컨설팅 덱 같은 PPT 계열 케이스는 첫 호출 때 템플릿을 한 번 고릅니다.

```
/roasting 이번 분기 임원 PPT (실적 + 리스크 + 결정사항 3개)
```

이후 흐름:
1. `/roasting`이 케이스를 `p41` (임원용 PPT)로 분류합니다.
2. `slide_library`에서 색상과 격식 기준으로 후보 템플릿을 추천합니다.
3. 그중 하나를 고르면 BLACK이 그 템플릿에 맞춰 HTML 덱을 짓습니다.
4. 4명의 평가가 끝나면 최종 HTML 파일이 남습니다.

## 7. 알려진 한계

**v0.1 베타 기준:**

- **베타 라벨:** v0.2에서도 케이스 품질과 평가 기준을 계속 다듬어 갑니다.
- **Agent Teams 실험성:** 평가자 4명이 동시에 도는 경로는 Anthropic 실험 기능을 씁니다. 오류가 나면 순차 모드로 자동 전환됩니다.
- **자동 sync 미지원:** 5color 사이트가 업데이트되면 `make sync-cases`를 한 번 돌리면 됩니다.
- **한국어 전용:** 케이스와 평가 기준 모두 한국어로 작업합니다. 영어 입력 최적화는 아직 없습니다.
- **HTML 슬라이드:** 현재는 HTML 형식으로만 지원합니다. PowerPoint(.pptx) 직접 출력은 v0.2에서 추가됩니다.
