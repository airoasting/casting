# roasting — 임원용 가이드

> `/roasting`은 화이트칼라 산출물을 5명이 함께 다듬는 Claude Code 플러그인입니다.
> 이메일부터 이사회 보고서까지, 63개 케이스를 자동으로 분류하고 합격선 9.5를 목표로 작동합니다.

---

## 1. 시작 (5분)

**설치 (두 줄):**

```bash
/plugin marketplace add airoasting/roasting
/plugin install roasting@airoasting
```

**첫 호출 예시:**

```
/roasting 거래처 부장한테 8월 12일까지 회신 달라는 메일
```

그러면 `/roasting`이 자동으로 가장 맞는 케이스를 찾고, 담당 역할로 캐스팅된 5명이 함께 작업을 시작합니다.
결과물은 Claude Code 대화창에 바로 나타납니다.

---

## 2. 이게 무엇인가

세 가지 시스템이 합쳐져 작동합니다.

| 구성 요소 | 역할 |
|---|---|
| **5color 사이트** | 63개 케이스 정의 원본 (airoasting.github.io/5color) |
| **slide_library** | PPT 케이스용 슬라이드 템플릿 35종 |
| **/roasting** | 자동 분류 + 5인 팀 실행 엔진 |

사용자가 자연어로 요청하면 `/roasting`이 63개 케이스 중 가장 잘 맞는 것을 찾습니다. 그 케이스 정의에 맞게 역할이 자동으로 정해지고, 5명이 각자 맡은 관점으로 산출물을 만들고 평가합니다.

## 4. 5-Color 방법론

5명이 한 산출물을 함께 만들고 평가합니다.

| 역할 | 담당 관점 | 한 줄 |
|---|---|---|
| **BLACK** | 수행자 | 케이스에 맞는 전문가 페르소나로 산출물을 직접 작성합니다 |
| **RED** | 이성·논리 | 의도가 한 줄로 전달되는지, 논리 구조가 단단한지 평가합니다 |
| **SILVER** | 분야 전문가 | 해당 문서 유형의 구조와 관행에 맞는지 확인합니다 |
| **BLUE** | 공감·톤 | 읽는 사람 입장에서 톤과 분량이 적절한지 봅니다 |
| **GOLD** | 독자 시나리오 | 실제 수신자가 이 문서를 받는 순간을 시뮬레이션합니다 |

**합격 기준:**
- RED·SILVER·BLUE 3인 평균 9.5 이상
- GOLD 체류도 9.5 이상
- 최대 4라운드 반복

4명의 평가자 중 점수 차이가 0.5 이상 벌어지면 자동으로 토론이 열립니다. 토론 후 새 점수를 반영해 다음 라운드를 진행합니다.

---

## 5. PPT 케이스 — slide_library 결합

임원 PPT, 영업 덱, 컨설팅 덱 같은 PPT 케이스는 첫 호출 시 템플릿을 확인합니다.

```
/roasting 이번 분기 임원 PPT (실적 + 리스크 + 결정사항 3개)
```

이후 흐름:
1. `/roasting`이 케이스를 `p41 (임원용 PPT)`으로 분류합니다.
2. `slide_library`에서 색상·격식 기준으로 가장 맞는 템플릿 후보를 보여줍니다.
3. 사용자가 템플릿을 선택하면 BLACK이 HTML 덱으로 작성합니다.
4. 4인 평가 후 최종 HTML 파일이 저장됩니다.

## 6. 베타 피드백

사용하다 불편한 점이 생기면 Claude 대화창에서 바로 피드백을 보낼 수 있습니다.

```
/roasting --feedback
```

세션 메타데이터가 자동으로 첨부된 GitHub Issue 작성 화면이 열립니다. 내용(산출물·입력)은 포함되지 않습니다.

---

## 7. 알려진 한계

**v0.1 베타 기준:**

- **베타 라벨:** 케이스 품질과 평가 기준은 v0.2에서 지속 개선됩니다.
- **Agent Teams 실험성:** 4명의 평가자가 동시에 실행되는 경로는 Anthropic의 실험 기능을 사용합니다. 오류 발생 시 자동으로 순차 실행 방식으로 전환됩니다.
- **자동 sync 미지원:** 5color 사이트가 업데이트되면 `make sync-cases`를 수동으로 실행해야 합니다.
- **한국어 전용:** 모든 케이스 정의와 평가 기준은 한국어로 작성되어 있습니다. 영어 입력에 대한 최적화는 이루어져 있지 않습니다.
- **슬라이드 출력:** HTML 형식으로 제공됩니다. PowerPoint(.pptx) 직접 출력은 v0.2 예정입니다.
