# 검증된 팀 레시피(하네스) + 라우터 규칙

> 목적이 아래 레시피에 맞으면 그대로 쓰고, 아니면 카탈로그에서 새로 조합한다. (정본 데이터: `docs/assets/router.js`. 부품은 회사형 10개 본부.)

## 카테고리
0. 리서치·인텔리전스 / 1. 재무·숫자 / 2. 전략·기획 / 3. 문서·커뮤니케이션 / 4. 생산성·운영

## 토글(모디파이어)

| 토글 | 켜는 신호 | 추가/효과 |
|---|---|---|
| 사실검증 | 수치·주장·근거 중요 | research 단계 뒤 팩트·출처 검증가(7) (진위+원출처 신뢰도) |
| 분석 강화 | "심층/깊게/자세히" 요청 | 레시피면 지정 deepAdd 1명. **새 조합이면 결정론 사다리로**: 다룰 데이터·수치가 있으면 데이터 분석가(10), 리스크·의사결정 성격이면 리스크 분석가(16), 둘 다 아니면 켜지 않는다. 추가 인원이 이번 입력에 쓸 자료가 없으면 넣지 않는다 |
| 반박 검토 | **미결 판단을 요구하는 말**("검토/결정/추천/할까 말까/어느 쪽/대응 전략")이 있을 때 | 게이트 **앞 실행 단계**에 비판적 검토자(47). 게이트 자리에는 넣지 않는다 |
| 영문/번역 | 영어 산출물·해외 대상 | 마지막에 번역·현지화 담당(35) |

## 라우터 규칙 (결정 사다리)

목적을 받으면 위에서부터 순서대로 판정한다. 재현성(같은 입력 → 같은 팀)을 위해 주관 서술어 대신 이 사다리를 따른다.

1. **하한 먼저**: 산출물이 1종·1페이지 이하·단일 역할로 쓸 수 있으면(공지 한 장, 메일 한 통, 단발 정리) 레시피를 무시하고 **작성 1 + 게이트 검토 1**로 고정한다. 워크스페이스·토글을 켜지 않는다.
2. **레시피 매칭**: 매칭 키워드가 걸린 레시피를 후보로 모은다. 걸린 키워드 수가 많은 것을 고른다. 동수면 목적의 **최종 산출물 종류**와 레시피의 마지막 산출 단계가 일치하는 것을 우선한다. 그래도 갈리면 3으로 간다.
3. **새 조합**: 맞는 레시피가 없거나 갈리면 `catalog.md`에서 *리서치/입력 → 분석/가공 → 작성 → 검토* 모양으로 조립한다. 겹치는 역할은 아래 경계 규칙으로 가른다. 산출물이 2종 이상이고 부품이 안 겹치면 산출물별 미니 파이프라인 + 공통 리서치 1명 공유 + 게이트 1명 합류.
4. **게이트 검토자 선택**: 계약·법무·문서 = 문서·품질 검수(46), 짧은 글·메일·공지 = 교정·윤문 담당(34), 규정·준법·감사 = 감사인(45), 그 외 일반도 문서·품질 검수(46). 숫자·재무면 재무 분석가(12) 교차검증 후 게이트. **비판적 검토자(47)는 게이트에 두지 않는다**(반박은 실행 단계에서).

## 겹치는 역할 경계 (tie-break)

새 조합에서 후보가 겹치면 한 질문으로 가른다.

| 겹치는 역할 | 결정 질문 | 왼쪽 | 오른쪽 |
|---|---|---|---|
| 카피라이터(22) ↔ 교정·윤문(34) | 새로 쓰나, 기존 문장을 다듬나? | 새로 씀→카피 | 다듬음→교정·윤문 |
| 콘텐츠 마케터(18) ↔ SEO·GEO(19) | 주제·서사인가, 노출 최적화인가? | 콘텐츠 | SEO·GEO |
| 데이터 분석가(10) ↔ KPI 추적(15) | 지표가 정의돼 반복 추적인가, 탐색인가? | 탐색→데이터 | 추적→KPI |
| 문서·품질 검수(46) ↔ 비판적 검토(47) | 형식·완성도인가, 논리 반박인가? | 품질→검수 | 반박→비판적 |
| 문서·품질 검수(46) ↔ 감사인(45) | 일반 품질인가, 규정·통제 준거인가? | 품질→검수 | 준법→감사인 |
| 회계사(17) ↔ 재무 분석가(12) | 세무·회계 처리인가, 경영 분석인가? | 세무→회계사 | 분석→재무 |
| 변호사(44) ↔ 문서·품질 검수(46) | 법률 리스크인가, 일반 완성도인가? | 법률→변호사 | 품질→검수 |
| 이메일·CRM(23) ↔ 이메일·업무 서신(32) | 다수 캠페인 메일인가, 1:1 서신인가? | 캠페인→CRM | 1:1→서신 |
| 소셜(20) ↔ 퍼포먼스·광고(21) | 오가닉 운영인가, 유료 광고인가? | 오가닉→소셜 | 유료→퍼포먼스 |
| 경영전략가(1) ↔ 신규사업(2) ↔ 타당성(3) | 방향인가, 발굴인가, 검증인가? | 방향→경영전략가 | 발굴→신규사업 / 검증→타당성 |

디자인본부(25~29)는 gpt-image 프롬프트 + OpenAI 이미지 API로 실제 이미지를 산출한다. 전문가(17·42·44·45)는 실무 초안·1차 검토 보조이며 자문이 아니다.

## 검증된 레시피

### 리서치 보고서 완성
- 분야: 리서치·인텔리전스 · 주제 조사부터 검토까지, 신뢰할 수 있는 보고서 한 편.
- 팀장 지침: 리서치 보고서를 만듭니다. 조사 → 요약 → 작성 → 교정 순으로 진행하고, 각 단계 산출물을 다음 팀원에게 넘깁니다.
- 단계:
  1. 리서치 어시스턴트(6) — 주제 → 조사 메모
  2. 요약·브리핑 담당(33) — 조사 메모 → 핵심 요약
  3. 보고서 작성가(30) — 요약 → 보고서 초안
- 검토: 교정·윤문 담당(34) — 초안 → 다듬은 최종본
- deepAdd: 데이터 분석가(10) · 가능 토글: fact,deep,rebut,trans
- 키워드: 리서치,조사,보고서,리포트,report,research,조사보고

### 시장·경쟁 분석
- 분야: 리서치·인텔리전스 · 시장 지형과 경쟁사를 읽고 전략 인풋을 만듭니다.
- 팀장 지침: 시장과 경쟁을 분석합니다. 시장 조사 → 경쟁 분석 → 트렌드 → 보고서 순으로 엮습니다.
- 단계:
  1. 마켓 리서처(8) — 주제 → 시장 동향
  2. 경쟁·동향 분석가(9) — 시장 → 경쟁 지형
  3. 트렌드·인사이트 분석가(11) — 지형 → 트렌드 신호
  4. 보고서 작성가(30) — 분석 → 보고서 초안
- 검토: 문서·품질 검수(46) — 초안 → 검수 통과본
- deepAdd: 리스크 분석가(16) · 가능 토글: fact,deep,rebut,trans
- 키워드: 시장,경쟁,경쟁사,트렌드,산업,market,competitor,동향,분석

### 데이터 기반 의사결정
- 분야: 리서치·인텔리전스 · 데이터에서 선택지를 뽑아 근거 있는 결정을 돕습니다.
- 팀장 지침: 데이터로 의사결정을 돕습니다. 데이터 분석 → 선택지 비교 → SWOT → 보고서.
- 단계:
  1. 데이터 분석가(10) — 데이터 → 패턴·인사이트
  2. 사업 타당성 분석가(3) — 인사이트 → 선택지 비교
  3. 트렌드·인사이트 분석가(11) — 선택지 → 강약점 정리
  4. 보고서 작성가(30) — 정리 → 결정 보고서
- 검토: 문서·품질 검수(46) — 초안 → 검수 통과본
- deepAdd: 리스크 분석가(16) · 가능 토글: deep,rebut,trans
- 키워드: 데이터,의사결정,결정,지표,decision,data,선택,근거

### 재무 현황 리뷰
- 분야: 재무·숫자 · 재무제표를 읽어 이사회용 현황 보고로 묶습니다.

- 단계:

  4. IR·투자자 리포트(14) — 지표 → 이사회 보고
- 검토: 문서·품질 검수(46) — 보고 → 검수 통과본
- deepAdd: 리스크 분석가(16) · 가능 토글: deep,rebut,trans
- 키워드: 재무,현황,비용,KPI,지표,financial,실적,결산,손익

### 예산·재무 모델링

- 팀장 지침: 예산과 재무 모델을 설계합니다. 예산 → 모델링 → 리스크 → 투자자 보고.

  3. 리스크 분석가(16) — 모델 → 위험 평가
  4. IR·투자자 리포트(14) — 평가 → 투자자 보고
- 검토: 문서·품질 검수(46) — 보고 → 검수 통과본

- 팀장 지침: 임원 발표자료를 만듭니다. 슬라이드 구성 → 발표 스크립트 → 핵심 요약 → 교정.

  3. 요약·브리핑 담당(33) — 대본 → 핵심 메시지
