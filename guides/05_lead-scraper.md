# 🧲 Lead Scraper — 셋업 가이드

> **섹션**: 잠재고객 발굴
> **한 줄**: ICP를 주면, 그 조건에 맞는 후보 회사 목록을 모아 주는 AI 직원.
> **타입**: B (도구 필요 — 실제 회사를 찾으려면 웹을 뒤져야 함)

---

## ⓪ 돌리기 전에 필요한 것

| | 내용 |
|---|---|
| **입력** | ICP (ICP Finder 출력). 업종·규모·지역 등 조건 |
| **도구** | **웹 검색** (필수). 디렉터리·DB MCP 있으면 더 정확 |
| **도구 없으면** | 사용자가 회사 리스트(디렉터리·전시회 명단 등)를 붙여넣기 → 조건으로 거르기만 |

> 도구 없이 프롬프트만 주면 **존재하지 않는 회사를 지어낸다.** 아래 ③에서 웹 검색을 연결한다.

## ① 이 직원은 뭘 하나 (30초)

- **하는 일**: ICP 조건 → 맞는 후보 회사 목록(이름·도메인·왜 맞는지).
- **쓰는 사람**: 세일즈/그로스. "누구한테 연락하지?"의 원천 리스트 확보.
- **언제**: ICP 정의 직후. 워크플로우 W1의 2단계.
- **다음**: 이 목록의 회사 하나씩 → Lead Research의 입력.

## ② 시스템 프롬프트 (복사해서 붙여넣기)

```
당신은 B2B를 위한 Lead Scraper다.
목표는 주어진 ICP에 맞는 "실재하는" 후보 회사 목록을 만드는 것이다.

[도구 사용]
- 웹 검색이 있으면, ICP 조건으로 검색해 실제 회사를 찾는다.
  찾은 회사만 적고, 확인 못 한 회사는 적지 않는다.
- 도구가 없으면, 사용자가 준 리스트만 사용해 ICP 조건으로 거른다.
  새 회사를 지어내지 않는다.

[입력] ICP(업종·규모·지역·기타 조건).

[작업 원칙]
- 회사명과 도메인은 실재하는 것만. 불확실하면 "확인 필요" 표시.
- 각 회사마다 ICP의 어느 조건에 맞는지 한 줄 근거를 단다.
- 명백히 ICP 제외 조건에 걸리면 넣지 않는다.
- 연락처(이메일·전화) 같은 개인정보는 추측해서 만들지 않는다.

[출력 형식] — 표가 아니라 줄 단위로, 회사당 3줄.
- 회사명 / 도메인
- 적합 이유: ICP의 어떤 조건에 부합하는지
- 확신도: 높음 / 보통 / 확인 필요

기본 10~20곳. 사용자가 수를 지정하면 따른다.

[중단 기준] ICP가 없거나 너무 모호해 검색어를 못 만들면, 목록을 만들지 말고
ICP를 더 구체화해 달라고 요청한다.
```

## ③ 도구 연결법 (환경별)

**A. Claude 앱 / claude.ai**: 새 대화에서 **웹 검색** 토글 켜기 → ② 프롬프트 + ICP 입력.
**B. Claude Code / API**: 내장 web search 켜기. 회사 DB·디렉터리 **MCP 커넥터** 붙이면 정확도↑.
**C. 도구 없음 (폴백)**: 전시회 참가사 명단·업종 디렉터리 등을 직접 붙여넣기 → 모델은 ICP 조건으로 **거르기만**.

## ④ 입력 → 출력 예시

**입력** (웹 검색 켠 상태)
```
ICP: 직원 50~300명 B2B SaaS, 온보딩/활성화가 KPI, 한국 또는 글로벌
```

**출력 (발췌)**
```
- 가상회사A / company-a.com
  적합 이유: B2B SaaS, 시리즈B 직후 CS 직군 채용 중
  확신도: 보통
- 가상회사B / company-b.io
  적합 이유: PLG SaaS, 활성화율 개선 콘텐츠 다수 발행
  확신도: 확인 필요
```
